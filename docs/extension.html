Extensions

  • One The most exciting feature of v2 is Extensions. it allows you to extend tagscript without messing with the internal code, or allows you to add 3rd party extensions too.

Registering a Extension

To Register a Extension you must call register() method of the Parse

Example:

parser.register(extension, { someOption: true, log: true })

First parameter of extension must be a function
Second Parameter will be the options for the extensions

  • Internally extension (1st parameter) will be called with options (2nd parameter) as the parameters

Example 2: A Fully working example may be found in ./examples directory here

Creating a Extension

register function expects a function to call as it first parameter

  • Its Suggested to export a function from your main extension file (if its a 3rd party public extension)
  • even a private extension needs to export a function : )
  • You can have a class as extension but export a function (if you want you can use this to make it easy (utility builtin) createLoadExtensionData)

Example of using createLoadExtensionData is here
Fully working extension is here

  • Name is a mandatory field
  • There is 2 parts to a Extension
    • InterpreterExtensions - Parts that Extends / Interact the Interpreter
    • StaticExtensions - Parts that Extends / Interact with the static Data Generator
    • Both most likely will be used together to extend tagscript syntax related or other parts

Simple Example

extension.js:

const { createLoadExtensionData }  =  require('@typicalninja21/tagscript');

// import { createLoadExtensionData } from '@typicalninja21/tagscript'

 class Extension {
		constructor(options) {
			this.options = options
		}
}

function loadExtension(options) {
	// options is passed as a param
	const ext = new Extension(options)
	return createLoadExtensionData(ext)
}

module.exports = loadExtension;
// export default loadExtension (typescript)
