Functions

You can call functions as demonstrated below

  • The Important part to call the Function is “()”

  • make sure there no spaces between “function name” and “()”

  • if throwError is enabled and there is no function name that, a error will be thrown

  • by default there are no functions defined

  • And you (as the user writing script) cant define functions

Functions without params

functions without params are called normally like below

function()

  • A Function without anything in between “()” is a Function without params

  • Passing no params to a function that expects param’s might throw a Error (depends on the function)

Functions with params

Functions with params are bit different but not complex

function(<param>, <param>)

  • Define functions params inside “()”

  • separate each param by a “,”

  • strings should be between acceptable string start and ends characters (", ')

  • functions in param’s is Supported but is not guaranteed to be stable and fast

Variables

see the below sub topics on different ways of using them

  • if throwError is enabled and variable is not in the specified context, error will be thrown

  • “.” syntax is supported (ex: variable1.variable2 < Variable 2 must be a Sub Property of variable1)

Defining a variable

Format to defining a variable is variable = value

variable = "string value"

# or

variable = functionValue();

# or

variable2 = variable

View Example

  • Value can be a string / function / another variable

  • when defining you may override a previously defined variable

  • make sure to define a variable before using it

  • If variable is undefined and throwError is enabled it will throw a Error

“.” syntax

“.” syntax is simple, see the below example


variable1.variable2

  • Here, variable 1 has a sub property called variable2.

  • You can keep chaining these (only if the sub Properties exists)

  • if throwError is enabled and variable1 is undefined a Error will be thrown

Conditions (if, else if, else)

Conditions are of 3 parts, those are (displayed in how they need to be written in),

  1. if - Every condition chain MUST Start with a IF. it MUST be at the start of the condition chain
  2. elseIF - elseIF’s are optional, and is ran if above if returns false for a condition
  3. else - else is also optional, how ever else’s MUST only be at a end if condition chain

For a condition to be triggered the condition must return true if the condition results in a false it either skips to elseIF or else (if present) or skips the condition block all together

Supported Conditions

  1. == - [Equal] :- used to check if 2 values are equal to each other, similar to JavaScript equality. returns Boolean (true/false)
    Example:

    if(1 == "1") // true
if(y == d) // false (y = "hey", d = "another")

  2. === - [Strict Equal] :- similar to above Equal , however it also checks for the type of the 2 values, similar to javascript strict equality. returns Boolean (true/false)
    Example:

    if(1 === "1") // false (different types, number | string)
if("1" === "1") // true

  3. > - [Greater than] :- works only on numbers (ex: l = 1). returns true if the left operand is greater than the right operand. similar to JavaScript greater than

  4. >= [Greater than or Equal] :- works only on numbers (ex: l = 1). returns true if the left operand is greater than or equal to the right operand, similar to JavaScript greater than or equal

  5. < - [Less than] :- works only on numbers (ex: l = 1). returns true if the left operand is less than the right operand. similar to JavaScript less than

  6. <= - [Less than or Equal] :- works only on numbers (ex: l = 1). returns true if the left operand is less than or equal to the right operand

code inside conditions

the code inside the condition is your general tagscript you use outside of it. however every new code block must be seperated by a ; (the parser splits code inside by ; and joins the splited parts with {code} so it can create a new Interpreter to run it)

Example:

{number2 = 2}
{string = "1"}
{result = "no data"}
{if(number2 > number):
	result = "number2 is greater than number 1";

	result = "string equals to 1";
